# ADR 001: Выбор основной базы данных

## Статус
✅ Принято

## Контекст
Приложению EcoTracker необходима база данных для хранения:
- Профилей пользователей
- Ежедневных записей о эко-активностях
- Каталога эко-советов и мероприятий

Критические требования:
1.  **Надёжность:** Минимальный downtime.
2.  **Геораспределённость:** Пользователи по всей России.
3.  **Бюджет:** Ограниченный стартовый бюджет.

## Рассмотренные варианты
1.  **PostgreSQL (open-source)**
    - Плюсы: Надёжность, ACID-транзакции, богатый функционал, бесплатно.
    - Минусы: Требует навыков администрирования.

2.  **MongoDB Atlas (managed)**
    - Плюсы: Готовая масштабируемость, управляемый сервис.
    - Минусы: Дороже, менее строгая схема данных.

3.  **Yandex Managed Service for PostgreSQL**
    - Плюсы: Полностью управляемый, интеграция с Yandex Cloud, георепликация.
    - Минусы: Vendor lock-in, дороже self-hosted.

## Решение
Выбрана **Yandex Managed Service for PostgreSQL**.

## Обоснование
1.  **Операционные расходы:** В команде нет dedicated DevOps на старте проекта. Управляемый сервис снижает операционную нагрузку.
2.  **Соответствие инфраструктуре:** Вся инфраструктура проекта разворачивается в Yandex Cloud. Использование их managed-сервисов упрощает развёртывание, мониторинг и безопасность.
3.  **Баланс цены и контроля:** Стоимость приемлема для бюджета MVP, при этом мы сохраняем мощь и надёжность PostgreSQL.

## Последствия
- Положительные: Быстрый старт, меньше времени на администрирование, встроенный бэкап.
- Отрицательные: Зависимость от вендора, стоимость выше, чем у self-hosted.
- Нейтральные: Для миграции потребуется 2-3 дня работы DevOps.
